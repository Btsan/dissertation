Different approaches to illustrating how approximate sketches estimate join cardinality.

Concept: Show the complete process vertically with explicit input formatting for each relation.

Concept: Minimize space while showing all key steps.

Summary & Recommendations

Key Elements to Show:

  1. Join query with filters on both relations
  2. Separate model inputs for each relation (formatted with hashes and [MASK])
  3. BERT inference for each relation produces probability distribution
  4. Sketch construction: probability × selection cardinality
  5. Dot product of sketches for join estimate

Best for Different Contexts:

  • Variation 1 (Vertical Flow): Most comprehensive, shows complete input tables
  • Variation 2 (Side-by-Side): Symmetric, emphasizes join operation visually
  • Variation 3 (Compact Pipeline): Best balance of detail and space efficiency
  • Variation 4 (Numbered Steps): Clearest progression for step-by-step explanation
  • Variation 5 (Flow Diagram): Most compact, shows parallel processing

Recommended Approach:

For the defense presentation:

  1. Primary slide: Variation 3 (Compact Pipeline) - good balance
  2. Alternative: Variation 4 (Numbered Steps) - if step-by-step clarity preferred
  3. Backup detail: Variation 1 or 2 - if committee wants more detail on input formatting
