How it Works

  • Input: Each attribute value is hashed to a bin ID (e.g., h(35) = 7). The join attribute is masked.
  • BERT Processing: The transformer uses bidirectional attention to condition on all known attributes.
  • Output: A probability distribution over hash bins for the masked attribute: P(h(customer_id) = i | age=35, city='SF', region='West')
  • Sketch Construction: Multiply the probability vector by the selection cardinality: sketch = p × |σ(T)|
Key Advantage: BERT can mask any attribute, allowing sketches to be approximated for arbitrary filter conditions at query time.
